¿Qué es el Prime Cube?

El Prime Cube es un robot construido con LEGO SPIKE Prime que puede resolver un cubo Rubik 3×3 de forma completamente autónoma. Analiza las caras del cubo usando el sensor de color, calcula la solución con un algoritmo matemático, y ejecuta los movimientos necesarios con motores grandes para resolver el cubo en cuestión de minutos.

Este proyecto combina construcción mecánica, programación avanzada y matemáticas. Es uno de los robots más complejos que pueden hacer con SPIKE Prime, y demuestra la potencia de la robótica educativa.

Componentes del Robot

El Prime Cube está construido con las siguientes piezas de SPIKE Prime. Cada componente tiene un rol específico en el funcionamiento del robot.

🧠
HUB SPIKE PRIME
El cerebro del robot. Ejecuta el programa y controla todos los componentes.
⚙️
2 MOTORES GRANDES
Giran las caras del cubo con precisión. Uno mueve el cubo, el otro lo rota.
🎨
SENSOR DE COLOR
Detecta los colores de cada cara del cubo para analizar el estado inicial.
🔧
ESTRUCTURA LEGO
Sostiene el cubo en posición y permite que los motores lo manipulen.

¿Cómo funciona?

El proceso que sigue el Prime Cube para resolver el cubo Rubik tiene tres etapas principales:

  • 1. ESCANEO: El sensor de color lee todas las caras del cubo, una por una. El robot gira el cubo para exponer cada cara al sensor y registra los colores en una matriz de datos.
  • 2. CÁLCULO: El programa analiza el estado del cubo y calcula la secuencia de movimientos necesaria para resolverlo. Usa algoritmos matemáticos como el "método de capas" o "método Kociemba".
  • 3. EJECUCIÓN: Los motores ejecutan los movimientos calculados: giros de 90°, 180°, en sentido horario y antihorario. Cada movimiento es preciso — el robot cuenta los grados que gira cada motor.

Todo esto sucede de forma completamente automática. El usuario solo coloca el cubo desordenado en la estructura, presiona el botón del Hub, y el robot hace el resto.

Pistas de Programación

Programar el Prime Cube es un desafío avanzado. Aquí hay algunas pistas de los conceptos principales que se usan en el código:

  • Variables y Listas: El programa guarda el estado del cubo en listas — arrays de 54 elementos (9 cuadrados × 6 caras). Cada elemento representa un color.
  • Funciones: El código está dividido en funciones reutilizables: escanear_cara(), girar_motor(), calcular_solucion(), etc. Esto hace que el programa sea más organizado.
  • Bucles (for/while): El escaneo de las caras usa bucles para repetir el proceso 6 veces (una por cara). Los movimientos también se ejecutan en bucles.
  • Condicionales (if/else): El programa toma decisiones según el estado del cubo: "Si esta cara está completa → pasar a la siguiente capa".
  • Control de Motores: Los motores se controlan con precisión usando run_to_position() para girar exactamente 90° o 180°. La velocidad y la aceleración se ajustan para que los movimientos sean suaves.
  • Algoritmos: La parte más compleja es el algoritmo de resolución. No es un código que inventemos nosotros — los métodos para resolver el cubo Rubik ya existen, y el robot implementa uno de esos métodos en código.

Nota: El código completo del Prime Cube puede tener cientos de líneas. No lo programamos desde cero en clase — usamos proyectos base que ya funcionan y los analizamos para entender cómo trabajan.
